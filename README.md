# Добавлялка треков в плейлист Я.Музыки

Вставляешь ссылку на трек в веб-форму — трек уходит в заранее заданный плейлист.

Работает через неофициальную библиотеку [`yandex-music`](https://github.com/MarshalX/yandex-music-api)
(официального открытого API для правки плейлистов у Яндекса нет).

## Как это устроено

- `core.py` — вся логика: разбор ссылки + добавление трека в плейлист.
- `web.py` + `templates/index.html` — веб-оболочка на Flask.
- `get_token.py` — получить токен.

Плейлист и токен принадлежат **одному** аккаунту (тому, кто создал плейлист).
Пользователи веб-формы ничего не логинят — все их треки падают в этот один плейлист.

## Установка

```bash
pip install -r requirements.txt
```

## Шаг 1. Токен

Токен — это доступ к твоему аккаунту Я.Музыки. Получить:

```bash
python get_token.py
```

Скрипт покажет ссылку и код (OAuth Device Flow). Открой ссылку, введи код,
подтверди вход — в консоли появится `access_token`. Сохрани его любым способом:

- в `.env` рядом с проектом (скопируй из `.env.example`) — подхватывается
  автоматически при запуске (`python-dotenv`), в том числе локально без Docker:
  ```
  YM_TOKEN=ПОЛУЧЕННЫЙ_ТОКЕН
  ```
- или через переменную окружения:
  ```bash
  export YM_TOKEN="ПОЛУЧЕННЫЙ_ТОКЕН"      # Linux / macOS
  set YM_TOKEN=ПОЛУЧЕННЫЙ_ТОКЕН           # Windows (cmd)
  $env:YM_TOKEN="ПОЛУЧЕННЫЙ_ТОКЕН"        # Windows (PowerShell)
  ```

Альтернатива без скрипта: открыть в браузере
`https://oauth.yandex.ru/authorize?response_type=token&client_id=23cabbbdc6cd418abb4b39c32c41195d`,
войти, и из адреса вида `https://music.yandex.ru/#access_token=...` скопировать значение
`access_token`. (Включи троттлинг сети в DevTools — редирект быстрый.)

Токен живёт примерно год. Своё OAuth-приложение Яндекс создать не даёт —
библиотека использует client_id официального клиента Я.Музыки.

## Шаг 2. Плейлист

Плейлист уже захардкожен в `core.py` (переменная `PLAYLIST`):
`https://music.yandex.ru/playlists/41cb8a47-dc72-3f22-a151-f6ea27c34464`

Поменять можно там же или через переменную окружения. Принимается любой вид:

```bash
export YM_PLAYLIST="https://music.yandex.ru/playlists/<uuid>"   # полный URL
export YM_PLAYLIST="41cb8a47-dc72-3f22-a151-f6ea27c34464"       # просто UUID
export YM_PLAYLIST="1003"                                       # старый числовой kind
```

Плейлист должен принадлежать аккаунту, чьим токеном пользуешься —
менять чужие плейлисты API не даст.

## Запуск

```bash
python web.py
# открой http://127.0.0.1:5000
```

Вставляешь ссылку в поле → трек в плейлисте. Дедупликация есть:
если трек уже в плейлисте, повторно не добавит.

Под формой показывается текущее содержимое плейлиста (название и исполнитель
каждого трека) — подтягивается через `GET /tracks` и обновляется после
каждого успешного добавления.

## Прод — Docker

`python web.py` поднимает dev-сервер Flask (только для локальной проверки).
В прод — через Docker: там gunicorn на `0.0.0.0:8000`.

Разово:

```bash
cp .env.example .env      # вписать YM_TOKEN (и при желании YM_PLAYLIST)
docker compose up -d --build
```

Приложение на `http://<сервер>:8000`. Проверка живости — `GET /healthz`.
Логи: `docker compose logs -f`. Остановить: `docker compose down`.

Без compose, если удобнее чистый docker:

```bash
docker build -t ym-adder .
docker run -d --name ym-adder -p 8000:8000 \
  -e YM_TOKEN="<токен>" \
  -e YM_PLAYLIST="https://music.yandex.ru/playlists/<uuid>" \
  ym-adder
```

Важное про секреты и прод:

- Токен передаётся только через переменную окружения / `.env`, в образ он не
  зашивается. Файл `.env` не коммить (он уже в `.gitignore`).
- Наружу лучше выставлять не голый 8000, а через nginx/Caddy с HTTPS.
  Тогда в compose порт можно оставить `8000:8000` и проксировать на него.
- Токен живёт ~год и кешируется в процессе. Когда протухнет — обнови `.env`
  и `docker compose up -d` (перезапустит с новым токеном).
- Плейлист общий: всё, что вводят пользователи, падает в один твой плейлист.
  Публичный доступ без какой-либо защиты = любой сможет накидать туда треков —
  если это нежелательно, поставь перед сервисом простой пароль (basic-auth в nginx).

## Оговорки

- API неофициальный: Яндекс может поменять его в любой момент, тогда библиотеку
  придётся обновить (`pip install -U yandex-music`).
- Не гоняй тысячи запросов подряд — можно словить ограничение аккаунта.
