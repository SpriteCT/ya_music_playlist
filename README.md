# Добавлялка треков в плейлист Я.Музыки

Любой человек логинится в свою Я.Музыку прямо в браузере (через Яндекс,
без пароля от этого сервиса) и создаёт ссылки вида «сбор треков на
вечеринку» — каждая ведёт в свой плейлист. Дальше ссылкой можно поделиться
с кем угодно: получатели просто вставляют ссылку на трек или альбом в
форму, без какой-либо регистрации — все треки уходят в плейлист владельца
ссылки.

Работает через неофициальную библиотеку [`yandex-music`](https://github.com/MarshalX/yandex-music-api)
(официального открытого API для правки плейлистов у Яндекса нет).

## Как это устроено

- `core.py` — вся логика: разбор ссылок, добавление треков/альбомов,
  список плейлистов аккаунта.
- `store.py` — маленькое SQLite-хранилище (`state.db`): пользователи
  (аккаунт Я.Музыки → токен, полученный через OAuth) и их шаринг-ссылки
  (slug → плейлист конкретного пользователя).
- `web.py` + `templates/*.html` — веб-оболочка на Flask:
  - `/login` — вход через Яндекс (OAuth Device Flow);
  - `/me` — личный кабинет: свои плейлисты, создание/удаление своих ссылок;
  - `/s/<slug>` — публичная форма добавления треков для конкретной ссылки.
- `get_token.py` — самостоятельная утилита для получения токена вручную
  (например, чтобы поиграться с `yandex_music` в консоли); веб-приложением
  не используется.

У каждого пользователя свой токен и свои ссылки — один не видит и не может
удалить чужие. Гости, переходящие по `/s/<slug>`, никак не авторизуются —
они просто добавляют треки в плейлист, привязанный к этой ссылке.

## Установка

```bash
pip install -r requirements.txt
cp .env.example .env
```

В `.env` нужен как минимум:

```
FLASK_SECRET_KEY=  # python -c "import secrets; print(secrets.token_hex(32))"
```

Без стабильного `FLASK_SECRET_KEY` сессии всех пользователей слетают при
каждом перезапуске процесса — для локальной проверки не страшно, для прод
обязательно задать.

## Запуск

```bash
python web.py
# открой http://127.0.0.1:5000 — описание сервиса,
# внизу ссылка «Войти через Яндекс»
```

1. Нажми «Войти через Яндекс» — покажется ссылка и код (OAuth Device Flow).
   Открой ссылку на любом устройстве, введи код, подтверди вход — страница
   сама обнаружит подтверждение (поллинг) и пустит в личный кабинет `/me`.
2. Выбери плейлист из своего аккаунта и нажми «Создать ссылку» — отдельное
   название вводить не нужно, ссылка называется как сам плейлист. Ссылку
   `/s/<slug>` можно скопировать и отправить кому угодно.
3. Плейлистов и ссылок может быть сколько угодно — под каждый повод/плейлист
   удобно завести отдельную ссылку. Ненужные — удаляются кнопкой в таблице.

### Публичная форма (`/s/<slug>`)

Сверху сразу видно, в какой плейлист уйдут треки — название плейлиста —
кликабельная ссылка на него в Я.Музыке. Вставляешь ссылку на трек → трек
в плейлисте. Дедупликация есть: если трек уже в плейлисте, повторно не
добавит.

Можно вставить и ссылку на альбом целиком (без `/track/…`, например
`https://music.yandex.ru/album/3192570`) — перед добавлением покажется
диалог со списком всех треков альбома (уже добавленные помечены «уже есть»),
и только после подтверждения все новые треки уйдут в плейлист.

Под формой показывается текущее содержимое плейлиста: название (кликабельное,
ведёт на плейлист в Я.Музыке) и список треков — обновляется после каждого
добавления. Есть поиск по списку (фильтр по названию и исполнителю, на
клиенте, без лишних запросов).

## Прод — Docker

`python web.py` поднимает dev-сервер Flask (только для локальной проверки).
В прод — через Docker: там gunicorn на `0.0.0.0:8000`.

Разово:

```bash
cp .env.example .env      # вписать FLASK_SECRET_KEY
docker compose up -d --build
```

`docker-compose.yml` монтирует `./data` как volume под `state.db` — без
этого токены пользователей и ссылки терялись бы при каждой пересборке образа.

Приложение на `http://<сервер>:8000`. Проверка живости — `GET /healthz`.
Логи: `docker compose logs -f`. Остановить: `docker compose down`.

Без compose, если удобнее чистый docker:

```bash
docker build -t ym-adder .
docker run -d --name ym-adder -p 8000:8000 \
  -v "$(pwd)/data:/app/data" \
  -e FLASK_SECRET_KEY="<секрет>" \
  ym-adder
```

Важное про секреты и прод:

- Секрет сессий передаётся только через переменную окружения / `.env`, в
  образ он не зашивается. Файл `.env` не коммить (он уже в `.gitignore`),
  как и `data/`/`state.db` (там токены всех пользователей).
- Наружу лучше выставлять не голый 8000, а через nginx/Caddy с HTTPS —
  особенно важно для `/me`, т.к. сессия держится на cookie.
- Плейлисты общие в рамках своей ссылки: всё, что вводят гости по
  конкретному `/s/<slug>`, падает в привязанный к ней плейлист. Сама ссылка
  не защищена паролем — доступ регулируется тем, кому она разослана.

## Безопасность

- Вход — только через настоящий OAuth Я.Музыки, отдельного пароля от этого
  сервиса нет. `/oauth/start` ограничен — 10 запросов с одного IP за 5 минут,
  чтобы сервис нельзя было использовать для спама device-code запросами в
  Яндекс. Ограничение живёт в памяти процесса, не переживает перезапуск и не
  общее между воркерами gunicorn — поднимает порог, но не строгая гарантия.
- Сессионная cookie — `HttpOnly` и `SameSite=Lax` всегда; `Secure` включается
  флагом `SESSION_COOKIE_SECURE=1` в `.env`, когда сервис стоит за HTTPS
  (иначе cookie в браузере не выставится вовсе на голом http — оставь
  выключенным для локальной проверки).
- Каждое изменение своих ссылок (`/me/links*`) проверяет владельца по
  `session["user_id"]` — один пользователь не может удалить или прочитать
  чужую ссылку, даже зная её slug.
- `state.db` (токены всех пользователей) — обычный файл SQLite без отдельного
  ограничения прав доступа на уровне приложения; полагайся на права файловой
  системы (в Docker — на владельца процесса `appuser`). Файл — общая точка
  отказа: у кого есть к нему доступ на чтение, тот получает токены всех
  пользователей сразу, не только одного.
- Ни один шаблон не отключает автоэкранирование Jinja — названия плейлистов,
  лейблы ссылок и т.п. безопасно показываются даже если содержат `< >` и т.д.

## Оговорки

- API неофициальный: Яндекс может поменять его в любой момент, тогда библиотеку
  придётся обновить (`pip install -U yandex-music`).
- Не гоняй тысячи запросов подряд — можно словить ограничение аккаунта.
